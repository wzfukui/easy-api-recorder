# easy-api-recorder

`easy-api-recorder` 是一个内部使用的 Chrome 扩展，用来把浏览器里的真实网络请求快速整理成可导出的 JSON / HAR，方便现场工程师和大模型继续生成 API 调用代码。

它的目标不是替代 DevTools，而是把现场常见流程从：

`F12 抓包 -> 复制请求 -> 手抄脚本`

压缩成：

`开始录制 -> 走业务流程 -> 挑选请求 -> 导出 -> 交给模型或工程师继续开发`

## 核心能力

- 选择当前标签页并开始录制
- 基于 `chrome.debugger` 附着到目标标签页
- 监听 `Network` 事件并重建请求生命周期
- 默认抓取 `Document`、`XHR`、`Fetch`
- 支持按需打开 `Script`、`Stylesheet`、`Image`、`Font`、`Media` 等资源类型
- 支持 URL 包含规则 / 排除规则
- 支持是否抓取二进制响应体
- 抓包结果支持搜索、类型快速筛选、颜色区分
- 支持把条目加入导出队列，再统一导出
- 支持直接删除无效抓包记录
- 支持查看请求头、响应头、请求体、响应体
- 支持导出标准化 JSON 和 HAR

## 适用场景

- 现场排查客户系统接口
- 从真实业务流程里提取 API 请求样本
- 给大模型提供结构化抓包结果
- 快速沉淀接口调用脚本或 SDK 代码草稿

## 使用方式

### 1. 加载扩展

1. 打开 `chrome://extensions`
2. 打开右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择当前目录：
   `/opt/code/chris/SuperBody/dev/easy-api-recorder`

### 2. 开始录制

1. 打开目标网页
2. 点击扩展图标
3. 在弹窗里确认当前标签页信息
4. 点击 `开始录制`

### 3. 执行业务流程

在目标标签页里完成登录、查询、提交等你真正想抓取的业务动作。

### 4. 查看结果

1. 回到扩展弹窗
2. 点击 `打开结果页`
3. 在结果页里查看已抓取内容

### 5. 筛选与整理

你可以在结果页里做这些事：

- 按资源类型快速筛选
- 按 URL、方法、状态码、MIME、错误文本搜索
- 隐藏已加入导出队列的条目
- 批量选择可见项
- 把有价值的条目加入导出队列
- 直接删除噪音数据

### 6. 导出

支持两种导出方式：

- 导出全部结果
- 只导出导出队列里的结果

导出格式：

- `JSON`：更适合大模型和内部工具继续处理
- `HAR`：更适合通用抓包格式交换

## 页面结构

### Popup 弹窗

- 显示当前标签页标题
- 显示录制状态
- 显示条目数量
- 显示当前抓包范围摘要
- 提供开始录制、停止录制、打开结果页

### 结果页

- 顶部概览卡片：会话、状态、已抓取、导出队列
- 抓包设置：默认折叠
- 工具栏：已抓取 / 导出队列、类型筛选、搜索
- 抓包列表：支持颜色区分、勾选、批量操作
- 右侧详情抽屉：查看头信息和 body

## 抓包规则说明

抓包判断顺序如下：

1. 先检查资源类型是否启用
2. 如果配置了包含规则，URL 必须命中至少一条包含规则
3. 最后执行排除规则
4. 一旦命中排除规则，就不会进入抓包结果

说明：

- 包含规则和排除规则都支持 `*` 通配符
- 每行一条规则
- 例如：
  - `*/api/*`
  - `*token*`
  - `*.png`
  - `*analytics*`

## 导出队列工作流

推荐按下面方式使用：

1. 先在 `已抓取` 里浏览所有请求
2. 把真正有价值的请求加入 `导出队列`
3. 把噪音记录直接删除
4. 最后在 `导出队列` 里统一导出

这样导给模型的数据会更干净，后续生成代码时干扰也更少。

## 技术实现

### popup.html / popup.js

- 读取当前标签页
- 展示基础状态
- 触发开始录制 / 停止录制
- 打开结果页

### background.js

- 附着和释放 `chrome.debugger`
- 监听 `Network` 事件
- 按 `requestId` 拼装请求记录
- 获取请求体和响应体
- 应用资源类型与 URL 规则过滤
- 管理 session 和 export queue
- 使用 `chrome.storage.session` 持久化轻量状态

### pages/recorder.html / pages/recorder.js

- 展示会话概览
- 编辑抓包设置
- 搜索、筛选、查看抓包列表
- 管理导出队列
- 查看右侧详情抽屉
- 导出 JSON / HAR

### lib/shared.js

- 统一资源类型定义
- 统一抓包配置归一化
- URL 规则匹配
- body 裁剪与格式处理

### lib/har-export.js

- 把标准化抓包结果转换成 HAR 1.2

## 当前限制

- 当前版本只支持同一时间录制一个标签页
- 响应体在 `loadingFinished` 之后再尝试获取
- 大响应体会被截断保存，避免 session 过重
- 二进制 body 默认不抓，需要显式开启
- 当前版本只负责抓取和导出，不负责重放请求
